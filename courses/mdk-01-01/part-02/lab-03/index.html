    Лабораторная работа №3

    Обновлена: 10.10.2021

    Тема

    Cписки контроля доступа.

    Цель работы

    Изучить принципы контроля прохождения сетевого трафика при помощи ACL-списков различных типов.

    Часть 1. Стандартные списки для IPv4

    Изучаемая схема

    Адресация устройств

    Устройство Интерфейс IP-адрес Шлюз по умолчанию
    R1 G0/1 192.168.10.1/24 —
    R1 G0/2 192.168.20.1/24 —
    R1 G0/0 10.0.0.1/30 —
    R2 G0/1 192.168.30.1/24 —
    R2 G0/0 10.0.0.2/30 —
    PC-A NIC 192.168.10.2/24 192.168.10.1
    PC-B NIC 192.168.20.2/24 192.168.20.1
    PC-C NIC 192.168.30.2/24 192.168.30.1

    Порядок выполнения работы

    1. Назначьте адреса в соответствии с таблицей.

    2. Проверьте доступность соседних устройств.

    3. Настройте маршрутизацию любым способом.

    4. Настройте на маршрутизаторе R? ACL-список.

      Правила для ACL

      • Пропускает трафик от всех узлов из сетей 92.168.10.0/24 и 192.168.20.0/24 ко всем узлам в сети 192.168.30.0/24.
      • В качестве номера списка доступа используйте N = свой порядковый номер в группе.
      R?(config)# access-list N permit 192.168.10.0 <шаблонная маска>
R?(config)# access-list N permit 192.168.20.0 <шаблонная маска>
R?(config)# access-list N deny any

    5. Примените ACL-список к подходящему интерфейсу в нужном направлении.

      R?(config)# interface g0/?
R?(config-if)# ip access-group N {in | out}

    6. На R? выполните команду:

      show access-list N

    7. Проверьте, есть ли доступ от PC-A и PC-B к PC-C.

    8. Настройте стандартный именованный ACL-список.

      Правила для ACL

      • Cписок должен разрешать доступ для трафика со всех узлов из сети 192.168.30.0/24 ко всем узлам в сети 192.168.10.0/24.
      • Доступ в сеть 192.168.10.0/24 должен быть разрешен только для узла PC-B.
      • Назовите этот список в формате ACL_ВашаФамилия (далее вставлять вместо acl_name).
      Шаблонная маска

      В общем случае вычисляется путем вычитания маски сети из 255.255.255.255. Может чередовать 0 и 1 в двоичном представлении.

      R?(config)# ip access-list standard <acl_name>
R?(config-std-nacl)# permit host 192.168.20.2
R?(config-std-nacl)# permit 192.168.30.0 <шаблонная маска>
R?(config-std-nacl)# end

    9. Примените ACL-список к подходящему интерфейсу в нужном направлении.

      R?(config)# interface g0/?
R?(config-if)# ip access-group <acl_name> {in | out}

    10. Проверьте настроенный ACL-список на R?, выполнив команду 

      show access-lists

    11. Проверьте работу настроенного ACL-списка, выполнив эхо-запрос с PC-С от PC-А.

    12. Внесите изменения в ACL-список.

      • Добавьте в схему еще одну сеть: настройте оставшийся интерфейс на R2 и присоедините к нему PC-D.
      • Назначьте на PC-D адрес из сети 172.16.0.0/16.

    13. Внесите изменения в протокол динамической маршрутизации.

    14. Выполните команду.

      show access-lists

      Обратите внимание на номера записей.

    15. Добавьте строку в конец именованного ACL-списка.

      Новое правило для ACL

      • Правило должно разрешать доступ к сети 192.168.10.0/24 от новой сети.
      R?(config)# ip access-list standard <acl_name>
R?(config-std-nacl)# ? permit <IP-адрес сети> <шаблонная маска>
R?(config-std-nacl)# end

    16. Проверьте настроенный ACL-список на R?.

      show access-lists

    17. Проверьте работу настроенного ACL-списка.

    Часть 2. Списки для IPv6 и VTY

    Изучаемая схема

    Схема та же, что в первой части.

    Адресация устройств

    Устройство Интерфейс IP-адрес Link-local адрес Шлюз по умолчанию
    R1 G0/1 2001:DB8:ACAD:A::1/64 FE80::1 —
    R1 G0/2 2001:DB8:ACAD:B::1/64 FE80::1 —
    R1 G0/0 2001:DB8:AAAA:1::1/64 FE80::1 —
    R2 G0/1 2001:DB8:ACAD:C::1/64 FE80::1 —
    R2 G0/0 2001:DB8:AAAA:1::2/64 FE80::2 —
    PC-A NIC 2001:DB8:ACAD:A::3/64 — FE80::1
    PC-B NIC 2001:DB8:ACAD:B::3/64 — FE80::1
    PC-C NIC 2001:DB8:ACAD:C::3/64 — FE80::1

    Порядок выполнения работы

    1. Включите маршрутизацию IPv6.

      ipv6 unicast-routing

    2. Назначьте адреса в соответствии с таблицей.

    3. Настройте статический маршрут по умолчанию на R1, который будет пересылать пакеты с неизвестными адресами на R2.

      ipv6 route ::/0 <интерфейс> <next link-local>

      Аналогично выполните настройку на R2.

    4. Проверьте доступность всех устройств.

    5. Настройте доступ по telnet.

      • Назначьте пароль для подключения по VTY на R1 и R2.

        R(config)# line vty 0 ?     // вместо ? часто ставится 4
R(config)# password <пароль>

      • Проверьте подключение по telnet к R2 со всех PC. На PC нужно ввести команду:

        rlogin <IP-адрес>

    6. Настройте ACL-список.

      • Cоздайте ACL-список.

        Правила для ACL

        • Доступ к маршрутизатору R1 через Telnet разрешён только для узлов из сети 2001:db8:acad:a::/64.
        R?(config)# ipv6 access-list <acl_name>
R?(config-ipv6-acl)# permit tcp 2001:db8:acad:a::/64 any eq ?

      • Отобразите созданный ACL-список.

        show access-lists

      • Примените ACL-список на линиях VTY маршрутизатора R1.

        R?(config-ipv6-acl)# line vty 0 4
R?(config-line)# ipv6 access-class <name> {in | out}

      • Проверьте, что доступ по telnet есть только у PC из выбранной в ACL сети.

    7. Отредактируйте ACL-список.

      • Добавьте произвольную запрещающую запись в существующий список.
      • Проверьте, что новое правило работает.

      • Проверьте статистику, используя команду:

        show access-lists
